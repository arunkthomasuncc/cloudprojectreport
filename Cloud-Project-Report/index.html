Job Recommendation System and Salary Predictor

Developed By: Aditya Kapri| Arun Kunnumpuram Thomas | Prajval Bavi| Ron Abhram Cherian



Introduction

In this project we have developed a job Recommendation system and salary predictor.Our system takes as input a user's description like his/her skills ,industries he/she is interested in,location preferences,type of job and suggests 10 jobs .Our system also predtics the range of salary which the user should expect.

The motivation behind this project are the problems which we faced while searching for jobs.Job searching is a very frustrating process and we find most of the jobs search results to be irrelevant to what users search. For e.g. user might not have the required job experience and yet the search result will give jobs which expects some kind of experience. To get away from that we want to give a tool which can help to smooth the process.

We are converting a user's information to a single vectore .Similary we have created a vector for each job we have in the dataset. For the purpose of recommendation we are calculating cosine similarity between user's profile vector and job vectors and recommending the top 10 jobs. For Predicting the salary range for the user we have developed a naive bayes classifier and a logistic regression classifier.

DataSet

We have taken the dataset from kaggle's "US jobs on Monster.com" kernal.It is a pre-crawled dataset, taken as subset of a bigger dataset (more than 4.7 million job listings) that was created by extracting data from Monster.com, a leading job board. The datset has 22k rows and 14 attributes.Each row is a job posting with features like job description, job title,location,organization,salary etc.In the dataset salary(yearly) is a continious valued feature.We are dividing it into 7 classes 0-30K,40-50K,50-60K,60-70K,70-90K and greater than 90K.A sample snapshot of the data is given below. Click here to download the data.

Data Preparation: We are workig with 4 features :job description,location,job type(full time or temprorary),organization.For converting words to vectors we have used word embeddings(Glove vectors for Word Representation developed in Stanford).Glove gives us 1 * 50 dimension vector for a word.For getting a single vector for a feature like job description which has multiple words: we are finding vectors for each word and adding them to get the resultant vector.If any word is not present in the Glove vector we are using the vector for 'unk' (unknown).

We wanted to assign different weights to different words(according to their importance) while forming a single vector for a document. In order to achieve that we have calculated the tf-idf score of each word and multipled its vector by tf-idf score.

Algorithms
Naive Bayes Classification

Our dataset has features in text format.Since naive bayes generally gives good results with text data we are using it to predict salary range.Here, we use the Bayes theorem, $$P(C|x) = {P(x|C)P(C) \over P(x)}.$$

  • P(C|x) is the posterior probability, where we find the probability of C, given the occurrence of x.
  • P(x|C) is the conditional probability, the probability of x, given the occurrence C.
  • P(C) is the prior probability, the probability of C on the whole dataset.
  • P(x) is the probability of x, where x represents each datapoint.

In this particular case, we can omit P(x), as it will be a constant denominator.So, the Bayes theorem for classification can be written as, $$P(C|x) = {P(x|C)P(C)}.$$ Since we have 4 predictor variables, and 7 class variable, the equation can be rewritten as, $$P(C_{k}|x_{i})=(\prod_{i}P(x_{i}|C_{k}))P(C_{k})$$ where k ranges from 0 to 6.

Here, we are multiplying the conditional probabilities of all the predictor variables and the prior probability of class variable. We need to calculate the above posterior probability for each class variable and compare them to determine which class has higher probability.

Conditional Probability
-------------------------------------------------
In order to form the conditional probability of each words in each class we have to prepare set of data.

We need to form a RDD which contains only the unique words in the whole corpus. This will be used for forming the conditional probability matrix for each unique words. To form the unique word RDD, In the map function, while reading each line from the input RDD we emit a tuple like (word,1) and in the reducer we grouped the words using reducebykey function. Also the number of unique words is calculated using count() function.

We have to form an RDD in a such a way that keys are the classes and values are the combined words of documents belongs to that class. In the mapper, while reading the RDD, we emit the class as key and list of words in that line as value. In the reducer we group the class keys and combined the words list. This RDD can be used to find the total number words in each class.

To find the no of occurence of words in each class , we are using the above RDD. We will be emitting the key value pair as (word#class,1) as the tuple. In the reducer we are combining the keys. So that for each word we will get the number of occurence in Each class. This RDD can be used to find the conditional probability of a word in a classes. To do that we are emitting the dividing the count of each words in class by total number of words in that class. So we got the conditional probability of words present in each class.

Now we have to find the conditional probability of each unique word and its conditional probability for each class. In Order to calculate the conditional probability of a word for each class we have defined a function callled wordClassProbailityCalculateUsingMap. First we are finding the sum of smoothing factor and count of the word in the class. We are storing it as the numerator. We are taking the smoothing factor as 1. For the denominator part, we are adding the number of words in the class and product of smoothing factor and no of unique words in the document. We are doing this for each class for a given word. Then we divide the numerator by denominator and take log of its value.So we got the conditional probability of a word in each classes. We are storing this results as an RDD and storing it as a text file in Amazon S3 Bucket.

Prior probability of classes
------------------------------------------
Prior probability of each class is calculated by emitting (class,1) by reading the input file RDD. In the reducer we add the number of 1 associated with a class by using reducebykey method.also we are dividing the total sum of 1 by total number of documents in that class. This result is stored in the Amazon S3 bucket.

Predicting the salary
------------------------------------------
In Order to predict the salary of a given resume,first we are reading the conditional probability matrix and prior probability matrix from the amazon s3 bucket as RDD. For each words in the resume, we are adding the conditional probability for each classes and multiplying it with class prior probability also. Among the these probabilities which class giving the maximum probability is the predicted salary class of the given resume.

Logistic Regression Implementation

Assuming independently identically distributed data, we can compute the likelihood as $$ P(T \mid x) = \prod_{n=1}^{N} \prod_{k=1}^{K} P(T = k \mid x_n)^{t_{n,k}} = \prod_{n=1}^{N} \prod_{k=1}^{K} y_{n,k}^{t_{n,k}} $$ To generate multiple probability outputs for each class, we consider the indicator output targets. $$ T = \begin{bmatrix} t_{1,1} & t_{1,2} & \cdots & t_{1, K} \\ t_{2,1} & t_{2,2} & \cdots & t_{2, K} \\ \vdots & & & \vdots \\ t_{N,1} & t_{N,2} & \cdots & t_{N, K} \\ \end{bmatrix} $$ where $t_{n,k}$ is 0 or 1 with only one 1 per each row. We define our error function as: $$ E(w) = - \ln P(T \mid x) = - \sum_{n=1}^{N} \sum_{k=1}^{K} t_{n,k} \ln y_{n,k}. $$ The update for the weight matrix is given by gradient descent as below: $$ w \leftarrow w - \alpha \nabla_w E(w). $$ We use softmax function for getting the probability of the class: $$ g_k(x) = P(T=k \mid x) = \frac{\kappa_k}{\sum_{c=1}^K \kappa_c} $$ where, $$ \kappa = f(x ; w) = X w. $$ The gradient comes out to be: $$\nabla_w E(w) = -\sum_{n=1}^{N} \Bigg( t_{n,j} - g_j(x_n)\Bigg) x_n$$ So the final update equation for weights is: $$ w_j \leftarrow w_j + \alpha \sum_{n=1}^{N} \Big( t_{n,j} - g_j(x_n)\Big) x_n. $$ 1. For every sample we need to calculate the gradient which can be done using map function.
2. The gradient is being calculated for each class(we have k classes) and we emit (k,v) pair of (class, gradient).
3. Then we will use reduceByKey to calculate the total gradients and update the weight matrix for the kth class.
4. The updated matrix is then broadcasted to all the partitions so that the new gradient calculated is always with the latest weight matrix.
5. We will finally check for convergence by checking if the new weight matrix has changed w.r.t the old weight matrix,
if it hasnt we break and the algorithm has converged else we will keep it running for n iterations.


K Means Implementation

The input for KMeans implementation is a comma separated csv with values of job description, organization and salary. We are putting this csv in the Amazon S3 bucket and reading it as RDD from spark program.

Using elbow method, for our dataset we have found out the number of clusters we need for KMeans as 5. So in our program we took the 5 sample records from RDD as the initial cluster points.

In the mapper, for each documents we find out the closest cluster point among the 5 clusters by calculating the euclidean distance between the point and cluster point and transform the values as (closest cluster center, point ) using lambda function. In the reducer for each cluster center we take the average of points associated with that cluster center using reduceByKey method. The average of these points will be the new cluster center. We are doing these steps for 100 iteration. Instead of doing the steps for a number of iteration, we also tried the till the difference between the sum of old cluster center points and new cluster center points become lesser than a threshold.


Cosine Similarity Implementation

First we computed the TF-IDF of each of the word w.r.t to each row(Job description as the document and dumped to an external file.) Then we converted all the words of the Job description in each of the row into its corresponding word vectors using Glove Vectors. We multiplied each word vector with a smoothing parameter and then multiplied each of them with their corresponding TF-IDF value.

Our input to computing Document similarity includes TF-IDF scores computed, resume file for which you need to find the similarity and finally the keywords you want to search for.
The Resume document will be converted to the word vector using Glove vector.
Our dataset is then

1.read,
2.converted to glove vector,
3.Multiply by tfidf

Then take the dot product of above computed word vector with resume word vector to return the similarity score.

Things we learnt

  • One of the biggest learning for us from this project is that not all the algorithms work on all the data.Logistic regression in our case is predicting only one class for all the test examples
  • In case of naive bayes class distribution or no of traning examples belonging from each class affects the model performance greatly.In our case we initialy strarted with 13 classes with some classes with very few records in them.For this data the model accuracy was not good(12%).Then we combine some classes in order to make sufficient examples in each class;this increases our model performance to 84%
